First launch on macOS

Read this first

o3 isn't signed with a paid Apple Developer certificate, so on first open macOS Gatekeeper will say "o3 can't be opened because Apple cannot check it for malicious software." That's expected — the app is open source and unmodified. Here's how to open it anyway.

A Via System Settings recommended
1 Drag o3.app into your Applications folder, then double-click it. macOS blocks the launch — click Done.
2 Open System Settings → Privacy & Security and scroll to the Security section.
3 You'll see "o3 was blocked to protect your Mac." Click Open Anyway, then confirm with Touch ID or your password.
4 o3 opens. macOS remembers your choice — every launch after this is normal.
B One Terminal command fastest

Prefer the terminal? After moving o3 to Applications, strip the quarantine flag and open it directly:

Terminal · zsh
$ xattr -dr com.apple.quarantine /Applications/o3.app
$ open /Applications/o3.app

This removes only the download-quarantine attribute macOS adds to files from the internet. It doesn't change the app itself.

Every release is built in public from the source on GitHub — you can verify or build it yourself. Once notarization is in place, these steps go away.
